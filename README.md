# 3 메인페이지 기능 구현

구체적인 메인페이지 구성
![image](https://github.com/user-attachments/assets/7d52acf9-1b02-4072-954c-06e791713f38)

1.상단 헤더 영역

로고 영역: 왼쪽 상단에는 사이트 브랜드 로고(Vroom)를 배치해 브랜드 인지도를 높였습니다.

검색 바: 중앙에는 차량 모델명, 브랜드 등을 검색할 수 있는 입력창을 배치하여 사용자 접근성과 정보 탐색 편의성을 강화했습니다.

로그인/회원가입 버튼: 우측 상단에 로그인 및 회원가입 버튼을 배치하여 접근성과 시인성을 확보했습니다.

탭 메뉴 구성: '차량구매', '차량용품', '견적', '차량정보', '고객지원' 등 주요 서비스 항목을 탑 메뉴로 구성하여, 모든 주요 기능에 빠르게 접근할 수 있도록 설계했습니다.

2.메인 비주얼(슬라이더) 영역

-슬라이드 배너: 화면 중앙 상단에는 시선을 끄는 대형 슬라이드 배너를 배치했습니다. React의 상태관리와 framer-motion 또는 swiper.js 등 라이브러리를 활용하여 좌우 전환 가능한 이미지 슬라이드 기능을 구현했습니다.

-CBTI CTA(Call to Action): "차에도 MBTI가?"라는 문구와 함께 CBTI(차량 성향 테스트) 페이지로 이동하는 링크 버튼을 삽입하여, 사용자의 참여를 유도하고 서비스 체험을 자연스럽게 연결하도록 했습니다.

-시각적 강조: 고성능 스포츠카 이미지(예: Corvette)를 배경으로 사용하여 다이내믹한 브랜드 인상을 줍니다.

3.브랜드 선택 섹션

-자동차 브랜드 리스트: 슬라이드 하단에는 BENZ, TESLA, HYUNDAI, KIA, CHEVROLET 등 주요 브랜드의 로고를 사용한 선택 버튼을 배치했습니다.

-각 브랜드 아이콘은 클릭 시 해당 브랜드 차량 목록 페이지로 이동할 수 있도록 라우팅을 구성했습니다

-브랜드 선택 UI는 반응형이며, Tailwind CSS를 통해 호버 및 선택 시의 스타일링도 적용되어 있습니다.

-사용자 경험 중심의 디자인: 버튼에는 둥근 카드 형태와 그림자 효과를 적용하여 현대적이고 깔끔한 시각적 인상을 줍니다.

4.반응형 UI 적용

-전체 메인 페이지는 Tailwind CSS의 반응형 유틸리티를 기반으로 구성되었으며, 데스크탑뿐만 아니라 모바일/태블릿 환경에서도 적절한 크기와 위치로 요소들이 재배치되도록 구현하였습니다.


# 4 CBTI 기능 구현

![image](https://github.com/user-attachments/assets/6cb51d5e-966a-4127-8c53-034d83406878)
![image](https://github.com/user-attachments/assets/9f1bf37a-8dfd-41c5-b2c0-c66ce190aef7)

1. 인트로 화면
   
페이지 진입 시 보여지는 시작 페이지는 '어떤 차를 사야 할지 고민되시나요?'라는 문구와 함께 사용자의 선택을 유도합니다.

시각적으로 미래지향적인 차량 내부 이미지와 'CBTI' 로고를 강조하여 기능의 정체성과 목적을 명확히 전달합니다.

시작하기 버튼을 누르면 실제 테스트가 시작되며, useNavigate()로 질문 페이지로 라우팅됩니다.

2. 질문 페이지
   
각 질문은 "고속도로 주행이 잦은 편인가요?" 등 실제 차량 선택에 영향을 줄 수 있는 라이프스타일/운전 습관 기반 문항으로 구성되어 있습니다.

**이분법적 선택 버튼(그렇다 / 아니다)**과 함께 **프로그레스 바(진행도)**를 시각적으로 구현하여 현재 응답 진행률을 명확히 안내합니다.

질문들은 JSON 또는 배열 기반으로 관리되어 재사용과 확장성이 용이하며, 상태 관리는 useState, useEffect로 처리합니다.

각 응답은 사전에 정의된 T/B, C/A, U/S, K/F 항목별 점수에 영향을 주며, 결과 계산에 반영됩니다.

3. 결과 페이지
   
모든 질문 응답이 완료되면 결과 코드(예: TCUF)가 출력된다.

각 단어의 의미로는 

T: 최신기술 지향

C: 편안함 선호

U: SUV 선호

F: 외제차 선호

K: 국산 선호

B: 기본 지향

S: 세단 선호

A: 스포츠성 지향


결과 하단에는 키워드 뱃지를 시각화하여 사용자의 성향을 명확히 요약해줍니다.

추가 버튼으로는 다시하기, 내 차 보기를 배치하여 재참여 유도와 차량 추천 서비스 연결을 동시에 구현하였습니다.

다시하기는 상태 초기화 후 첫 질문으로 이동하고, 내 차 보기는 추천 차량 목록 페이지와 연동됩니다.


# 5 차량구매페이지 기능 구현

1. 차량 카드 내 시승 신청 기능
   
‘시승 신청’ 버튼을 통해 사용자는 선택한 차량의 시승 예약을 진행할 수 있습니다.

클릭 시 차량 정보가 자동으로 주문서에 포함되며, 사용자 입력을 통해 배송지 정보 및 결제 수단을 입력할 수 있도록 구성하였습니다.

실시간 반응형으로 입력 필드에 따라 상태가 관리되며, 폼 유효성 검사 및 약관 동의 여부가 체크되어야 다음 단계로 진행됩니다.

2. 장바구니 기능
   
🛒 버튼을 클릭하면 해당 차량이 장바구니에 담기며, 장바구니 페이지로 진입하거나 다른 차량을 계속 선택할 수 있습니다.

장바구니 페이지에서는:

차량의 이미지, 모델명, 단가, 수량, 총 가격을 확인할 수 있습니다.

수량 증감 버튼 + / -를 통해 동적으로 수량을 조절할 수 있으며, 수량 변경 시 합계 금액도 자동 갱신됩니다.

개별 차량 삭제(X 버튼)도 가능합니다.

전체 상품 구매하기 버튼 클릭 시 → 결제 페이지로 연결됩니다.

3. 주문/결제 화면
   
장바구니에 담은 차량 정보를 기반으로 최종 주문 페이지가 구성됩니다.

주문 내역(모델, 수량, 총합), 배송지 정보(이름, 연락처, 주소), 결제 수단을 입력하도록 UI를 구성하였습니다.

결제 수단은 카드결제, 무통장입금 등 선택형 드롭다운으로 구현되어 있으며, 추후 API 연동 시 간편결제도 연계 가능합니다.

결제 동의 체크박스를 클릭해야 결제 진행이 가능하며, 최종 ‘결제하기’ 버튼을 누르면 주문 완료 프로세스로 이동합니다.


# 6 차량용품구매페이지 기능 구현

![image](https://github.com/user-attachments/assets/359f335c-45f5-4c6d-9af1-3931beaa7dac)


1. 제품 목록 UI
   
카드형 제품 배치를 통해 각 액세서리를 직관적으로 보여주며, 구성은 아래와 같습니다:

상품 이미지

상품명 (예: 에어컨 필터, 차량용 와이퍼, 워셔액 등)

주요 기능 설명: 제품마다 최대 2~3개의 특징/기능을 bullet list 형식으로 요약

가격 정보: 정규화된 원화 표기(8,000₩, 2만₩ 등)

구매 버튼 및 장바구니 추가 버튼 포함

차량 액세서리 태그 표시를 통해 향후 상품 카테고리 필터링에 활용

2. 검색 기능
   
상단에는 브랜드명 또는 제품명으로 검색 가능한 입력 필드를 배치해, 사용자가 원하는 용품을 빠르게 찾을 수 있도록 구현

입력값은 소문자/대문자 구분 없이 자동 필터링되며, 페이지 내 렌더링된 아이템 리스트가 즉시 갱신됩니다 (onChange + filter() 활용)

3. 스타일 및 UX
   
Tailwind CSS 기반으로, 일관된 디자인 시스템을 자동차 페이지와 동일하게 유지하여 사용자 혼란을 최소화했습니다.

각 카드는 반응형 레이아웃을 갖추고 있으며, 모바일에서도 2단 혹은 1단 그리드로 자연스럽게 재배치됩니다.

버튼 클릭 시 상품은 장바구니에 추가되며, 장바구니 페이지로 바로 이동하거나, 이어서 쇼핑을 계속할 수 있습니다.

4. 재사용 가능한 컴포넌트 구조
   
액세서리 상품 카드 역시 ProductCard 컴포넌트로 구성되어 있으며, props를 통해 다양한 상품 정보를 렌더링

장바구니, 주문/결제, 구매 완료 흐름도 자동차 판매 페이지와 동일한 전역 상태 관리 구조를 활용

1. 차량 모델 및 옵션 선택
   
브랜드 / 모델 / 트림 / 색상 선택:

드롭다운 형식의 선택창으로 사용자가 차량을 구성할 수 있으며, Hyundai, Avante, 어비스 블랙 펄 등 브랜드 및 색상 선택 가능

선택된 조건에 따라 차량 이미지도 동적으로 변경되며, 이미지 영역에는 3D 회전 기능(Play 버튼)이 포함될 수 있도록 설계

옵션 선택 체크박스:

사용자는 Sunroof, Leather Seats, Navigation 등 추가 옵션을 체크하여 구성 가능

옵션 선택 시 가격이 실시간으로 합산되며, 체크박스 상태에 따라 차량 총액이 동적으로 업데이트됨

2. 가격 상세 계산
   
기본 차량가, 옵션 합계, 차량가 총계를 명확하게 구분하여 표시

선택한 옵션에 따라 차량가 총액이 계산되며, 모든 숫자에는 천 단위 쉼표와 원화 기호가 적용됨 (Intl.NumberFormat 사용)

3. 결제 방식 및 납부 정보 안내
   
결제 방식은 일시불 또는 카드 중 택일 가능 (Radio 버튼)

하단에는 세금/등록비/총 구입비용 등 실질 구매 시 필요한 부대비용까지 자동으로 계산되어 표시됨

예: 취득세, 등록세, 총 구입 금액 등

4. 사용성 및 UX
   
전체 UI는 모바일과 데스크탑 반응형을 고려해 제작되었으며, 각 요소는 Tailwind CSS 기반의 카드 형태로 깔끔하게 배치

견적서 오른쪽 상단에는 고유 번호(NO. 961214)를 표시하여 개인 견적서 출력/저장 기능을 향후 연동할 수 있도록 구성

# 8 차량정보페이지 기능 구현

![image](https://github.com/user-attachments/assets/6b35a886-39bd-46b2-8538-074d48c8e276)
![image](https://github.com/user-attachments/assets/46d61ea4-2e88-444f-b52a-af3026a70cc9)

차량 정보 페이지에서는 브랜드별 차량 리스트가 카드 형태로 나열되어 있으며, 각 카드의 하단에는 차량 상세 정보 버튼이 있습니다.

해당 버튼 클릭 시, 선택한 차량의 세부 사양을 볼 수 있는 상세 정보 페이지로 이동합니다.

예시: HyundaiIoniq 카드에서 버튼 클릭 → HyundaiIoniq 상세 정보 페이지 렌더링

-차량 정보 페이지 구성 
카드 형태 UI:

차량 이미지

내장 색상, 연비, 배기량

주요 옵션 요약

가격 정보 (~,000만원)

각 차량마다 차량 상세 정보 버튼 포함 → 클릭 시 useNavigate()로 해당 차량 ID 또는 이름을 기반으로 상세 페이지로 라우팅

-상세 정보 페이지 구성
선택된 차량의 이름, 종류(예: 전기 세단), 가격(혜택 전/후), 연비, 배기량을 상단에 강조

우측에는 외장 색상 선택 드롭다운과 차량 이미지가 있고, 하단에는 내장 색상 선택 및 시트 이미지가 표시됩니다.

내장 색상 선택 시 상태 변경에 따라 하단의 시트 이미지가 실시간으로 업데이트됨

연비 단위는 차량 타입에 따라 자동 변환(km/L vs km/kWh)

# 9 고객지원 기능 구현

![image](https://github.com/user-attachments/assets/6c3e8919-ddc5-43d8-a155-442dd338d996)

1. 상담 신청 입력 폼
   
사용자에게 다음 입력 항목을 요구합니다:

이름 (사용자 실명 입력)

이메일 (연락 가능한 메일 주소, 형식 유효성 검사 포함)

휴대 전화번호 (010으로 시작하는 대한민국 전화번호 형식)

문의 제목 (예: 차량 견적 문의, CBTI 결과 관련, 구매 절차 등)

상담 내용 (자세한 문의 내용을 자유롭게 입력)

2. 입력 인터페이스
   
각 입력 필드는 placeholder 및 아이콘을 포함하여 직관적으로 설계됨

입력 오류 방지를 위한 **유효성 검사(Validation)**가 클라이언트 측에서 처리됨 (onSubmit 시 필수 입력 확인)

상담 신청하기 버튼 클릭 시 사용자 입력 데이터가 서버 혹은 이메일로 전송되도록 POST 방식 연동 가능 (현재는 프론트엔드 UI 중심)

3. 디자인 및 사용자 경험
   
카드형 레이아웃으로 가운데 정렬된 집중형 폼 UI 제공

Tailwind CSS 기반으로 둥근 테두리, 입력창 간 여백, 깔끔한 검정 버튼 등 UX를 고려한 시각적 구성

반응형 디자인이 적용되어 모바일에서도 폼이 안정적으로 출력됨

4. 기술 구현 요약
   
useState로 각 입력 필드 상태를 관리하고, handleSubmit 함수에서 데이터 수집 및 유효성 체크

향후 백엔드 API와 연동 시, Axios 또는 fetch를 사용하여 입력 데이터를 POST 방식으로 전송 가능

자동 응답 메시지, 문의 내역 저장 기능도 추가 개발이 용이한 구조로 설계
